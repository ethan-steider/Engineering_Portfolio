Modular Drone Platform for Undergraduate Research

Capstone • Aug 2025 – May 2026

System Overview

This project develops a modular, autonomous octocopter platform for environmental and climate research. Flight control is handled by a Pixhawk Cube Orange+, while onboard data collection and networking are managed by an NVIDIA Jetson running a server. Optical, thermal, and weather sensors can be operated individually or simultaneously, enabling flexible data collection for researchers.

Key Engineering Contributions

  • Restored autonomous flight capability by debugging GPS and telemetry failures, achieving reliable GPS lock with 12+ satellites.
  • Redesigned the Jetson Nano power system to eliminate brownouts and reduce boot time from over 10 minutes to under 2 minutes.
  • Implemented a modular sensing architecture supporting optical, thermal, and weather sensors with independent data logging.
  • Designed and tested a low-latency 2.4 GHz network communications link for real-time data transmission.

Hardware & Architecture

  • Flight Control: Pixhawk Cube Orange+ running ArduPilot with CAN-based GPS and MAVLink telemetry.
  • Compute: NVIDIA Jetson Nano (onboard data capture & communications) and Jetson Orin (offline AI processing).
  • Sensing: Optical camera (HDMI→USB), thermal camera (USB), and SCD-41 CO₂ / temperature / humidity sensor (I²C).
  • Communications: TP-Link CPE210 directional antenna relaying a 2.4 GHz hotspot for low-latency video and data.

Testing & Results

  • Completed multiple short-duration flight tests validating stable hover, GPS performance, and live video streaming.
  • Confirmed stable MAVLink telemetry after radio replacement and parameter reconfiguration.
  • Identified PID tuning requirements for longer autonomous flights, guiding next-phase testing.

Engineering Standards

  • IEEE 1937.1 – Payload interface and performance requirements for drone-mounted sensors.
  • IEEE 1939.1 – Framework for low-altitude airspace and UAV operations.
  • IEEE 802.11 – Wireless LAN standards used for onboard data transmission.

Jetson-Based Video Streaming & Ground Station UI

Capstone Subproject • Embedded Linux

System Overview

Built an onboard video streaming pipeline and browser-based ground station UI to view live optical/thermal feeds from the UAV in real time. The system runs on NVIDIA Jetson (embedded Linux) and supports live viewing, recording, and basic operator controls over a local network.

Key Engineering Contributions

  • Implemented a Python + Flask web server to stream live camera feeds to a ground-station browser interface.
  • Integrated USB camera sources (optical + thermal) and handled device enumeration / swap scenarios.
  • Added recording and data management features to support field data collection and later analysis.
  • Improved reliability by addressing power/boot and reconnection issues observed during testing.

Hardware / Software Stack

  • Hardware: NVIDIA Jetson Nano, USB cameras (optical + thermal), onboard networking hardware.
  • Software: Ubuntu/Linux, Python, Flask, OpenCV, HTML/CSS/JavaScript (browser UI).

Testing & Results

  • Validated live video streaming and ground-station viewing during ground and flight tests.
  • Demonstrated stable operator viewing of camera feeds with low end-to-end latency at short range.
  • Captured representative in-flight footage and UI screenshots for documentation and demonstrations.
